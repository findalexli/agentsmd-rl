# Fix Clang Warning in Kotlin Native CExport

## Problem

When compiling C++ code generated by Kotlin/Native's CExport feature for non-Unit Kotlin functions, Clang emits the following warning:

```
warning: non-void function does not return a value in all control paths [-Wreturn-type]
```

The generated C++ code contains a try-catch block where the `try` block has a `return` statement but the `catch` block calls `HandleCurrentExceptionWhenLeavingKotlinCode()`. This function never returns — it either rethrows the exception or terminates the program via `std::terminate()`. Because the function is declared with a return type but never returns a value in the catch path, Clang warns.

## Relevant Files

All three files that declare, define, or generate code for `HandleCurrentExceptionWhenLeavingKotlinCode`:

1. `kotlin-native/runtime/src/main/cpp/Exceptions.h`
2. `kotlin-native/runtime/src/main/cpp/Exceptions.cpp`
3. `kotlin-native/backend.native/compiler/ir/backend.native/src/org/jetbrains/kotlin/backend/konan/cexport/CAdapterApiExporter.kt`

## Notes

- The `RUNTIME_NORETURN` macro exists in the codebase (defined as `__attribute__((noreturn))` on GCC/Clang) — search for it to find examples
- `HandleCurrentExceptionWhenLeavingKotlinCode` never returns to its caller
- The function calls `std::terminate()` in its implementation

## Validation

Your fix should ensure that `HandleCurrentExceptionWhenLeavingKotlinCode` is marked with `RUNTIME_NORETURN` consistently across all three locations where it appears.
