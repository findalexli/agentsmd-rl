# Task: Fix Clang Warning in Kotlin Native CExport

## Problem

When compiling C++ code generated by Kotlin/Native's CExport feature for non-Unit Kotlin functions, Clang emits the following warning:

```
warning: non-void function does not return a value in all control paths [-Wreturn-type]
```

This warning occurs because the generated C++ code has a try-catch block where the `try` block contains a `return` statement, but the `catch` block doesn't. Instead, the catch block calls `HandleCurrentExceptionWhenLeavingKotlinCode()` which never actually returns (it either rethrows the exception or terminates the program via `std::terminate()`).

## Your Task

Mark `HandleCurrentExceptionWhenLeavingKotlinCode` with the `RUNTIME_NORETURN` attribute in all relevant locations so that Clang understands all control paths are handled and stops emitting the warning.

## Relevant Files

1. **kotlin-native/runtime/src/main/cpp/Exceptions.h** - The header file declaring the function
2. **kotlin-native/runtime/src/main/cpp/Exceptions.cpp** - The implementation file
3. **kotlin-native/backend.native/compiler/ir/backend.native/src/org/jetbrains/kotlin/backend/konan/cexport/CAdapterApiExporter.kt** - The compiler component that generates C adapter code

## Notes

- The `RUNTIME_NORETURN` macro expands to `__attribute__((noreturn))` on GCC/Clang
- Other functions in the codebase like `ThrowException` already use this attribute - look at how they're declared for reference
- The fix needs to be consistent across all three locations (declaration, definition, and generated code)

## Validation

Your fix should ensure that:
1. The function declaration in `Exceptions.h` includes `RUNTIME_NORETURN`
2. The function definition in `Exceptions.cpp` includes `RUNTIME_NORETURN`
3. The generated code in `CAdapterApiExporter.kt` includes `RUNTIME_NORETURN`
