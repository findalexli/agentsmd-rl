# Schema.getPropertySignatures crashes on Structs with `optionalWith({ default })`

## Symptom

In the Effect TypeScript library (`effect/Schema`), calling
`SchemaAST.getPropertySignatures(schema.ast)` on a `Schema.Struct` whose
fields use `Schema.optionalWith` with options that introduce a
**Transformation** AST node — most notably `{ default }`, `{ as: "Option" }`,
and `{ nullable: true }` — throws:

```
Error: Unsupported schema
schema (Transformation): (Struct (Encoded side) <-> Struct (Type side))
```

The same call works for plain Structs (no `optionalWith`), Refinement
wrappers, suspended schemas, and Unions. Only Transformation-producing
variants crash.

## Reproduction

```ts
import * as S from "effect/Schema"
import * as AST from "effect/SchemaAST"

const schema = S.Struct({
  a: S.String,
  b: S.optionalWith(S.Number, { default: () => 0 })
})

AST.getPropertySignatures(schema.ast) // throws "Unsupported schema (Transformation)"
```

## Expected behaviour

`getPropertySignatures` must return one `PropertySignature` per declared
field in the Struct, exactly as it does for the equivalent Struct without
`optionalWith`. For a Transformation AST node, the property keys/types
should be read from the **decoded (type) side** — the same convention
already used by `getPropertyKeys` for the same situation.

For the example above, the call must return two `PropertySignature`s
named `"a"` and `"b"`, both with `isReadonly === true`. The variant
`{ as: "Option" }` should also succeed and yield a `PropertySignature`
for `"b"` whose `type._tag === "Declaration"` (the `Option<number>`
declaration).

## Why this matters

`@effect/ai`'s `Tool.getJsonSchema()` and `LanguageModel.streamText`
internally call `getPropertySignatures` when building tool definitions.
Any tool whose parameter schema uses `optionalWith({ default })` (a very
common pattern) currently crashes during schema introspection.

## Scope

The fix lives in the `effect` package's schema-AST module. Existing
test cases in the repository for `getPropertySignatures` (plain Struct,
Refinement, suspend, Union, Class) must continue to pass.
