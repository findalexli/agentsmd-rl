# Add Workflow example to worker-with-resources fixture

## Overview

The `worker-with-resources` fixture in Cloudflare's workers-sdk monorepo demonstrates
various Cloudflare Workers platform features: KV storage, R2 buckets, D1 databases,
and Durable Objects. However, it is missing an example for **Cloudflare Workflows**,
a feature that allows defining multi-step, long-running operations with sleep/retry
semantics.

Your task is to add a basic Workflow example to this fixture.

## What to change

The fixture lives under `fixtures/worker-with-resources/`. It has three files that
need to be updated to add Workflow support:

### 1. Source code (`src/index.ts`)

Add a new exported class named **`MyWorkflow`** that extends **`WorkflowEntrypoint`**
from the `cloudflare:workers` module. The class should:

- Be parameterized with `Env` and a params interface named **`WorkflowParams`**
- The `WorkflowParams` interface should have a `name` field of type `string`
- Implement an async **`run`** method that receives `event` (typed as
  **`WorkflowEvent<WorkflowParams>`**) and `step` (typed as **`WorkflowStep`**)
- Inside `run`, extract `event.payload.name`, defaulting to `"World"` if absent
- Call **`step.do`** at least once — for example, to create a greeting string that
  incorporates the name
- Call **`step.sleep`** to introduce a delay (e.g., `"1 second"`)
- Call **`step.do`** a second time to produce a final result string
- Return the final result

Additionally, add the necessary imports from `cloudflare:workers`:

- `WorkflowEntrypoint` should be added to the existing value import alongside
  `DurableObject`
- `WorkflowEvent` and `WorkflowStep` should be imported as **type-only** imports
  (use `import type { ... }` syntax)

### 2. Wrangler configuration (`wrangler.jsonc`)

Add a **`workflows`** array to the configuration. It should contain a single
object with these fields:

- `binding`: **`"MY_WORKFLOW"`**
- `class_name`: **`"MyWorkflow"`**
- `name`: **`"my-workflow"`**

### 3. Type declarations (`worker-configuration.d.ts`)

The `Cloudflare.Env` interface needs a new property **`MY_WORKFLOW`** typed as
`Workflow<Parameters<import("./src/index").MyWorkflow["run"]>[0]["payload"]>`.

This is the type generated by running `wrangler types --no-include-runtime`, so
the type expression should match what Wrangler would produce for a Workflow binding.

## Constraints

- Do not remove or break the existing `MyDurableObject` class, the default export
  fetch handler, or any existing imports
- Use `import type` for type-only imports per the repository's TypeScript conventions
- Do not use `any` types — use explicit TypeScript types
- Do not use the non-null assertion operator (`!`)
