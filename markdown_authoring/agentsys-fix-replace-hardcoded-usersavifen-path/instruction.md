## Bug

Both `.kiro/skills/web-auth/SKILL.md` and `.kiro/skills/web-browse/SKILL.md` hard-code the original developer's home directory in every CLI command example:

```bash
node /Users/avifen/.agentsys/plugins/web-ctl/scripts/web-ctl.js ...
```

This path is specific to one machine. On any other user's system, every command in these skills silently fails with a "file not found" error because `~/.agentsys/` lives under a different home directory.

**Security aspect**: Committing an absolute home directory path exposes the local filesystem layout of the original developer's machine. It also prevents any contributor from running the examples without manually substituting the path.

## Fix

Replace all 76 occurrences of `/Users/avifen/.agentsys/` with `~/.agentsys/` — the portable, shell-expanded equivalent that works on macOS, Linux, and Windows (via Git Bash/WSL).

This matches the convention already documented in `meta/skills/maintain-cross-platform/SKILL.md`, which recommends `~/.agentsys/` as the canonical install path reference.
