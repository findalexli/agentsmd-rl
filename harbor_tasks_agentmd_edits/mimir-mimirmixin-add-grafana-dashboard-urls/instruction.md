# Add Grafana Dashboard URL Annotations to Mimir Mixin Alerts

## Problem

Mimir's Prometheus alerts (defined in the mixin's jsonnet/libsonnet files) fire with useful `runbook_url` annotations but lack direct links to the relevant Grafana dashboards. Operators investigating an alert must manually search for the right dashboard and then set filter variables (cluster, namespace) by hand. This slows down incident response.

## What Needs to Happen

Add optional `dashboard_url` annotations to Prometheus alerts generated by the Mimir mixin, so operators can navigate directly from a firing alert to the relevant Grafana dashboard with cluster labels pre-filled as dashboard variables.

The feature should be:
- **Configurable**: controlled by a new config field in `config.libsonnet` for the external Grafana URL prefix
- **Opt-out by default**: when the config field is empty, no dashboard URLs should be emitted
- **Reusable**: implement helper functions in `alerts-utils.libsonnet` that other alert files can call — one for generating the full dashboard URL (using `std.md5` for the dashboard UID, same as the dashboard definitions), one for per-alert annotation, and one for bulk-applying a default dashboard to all alerts in a component file
- **Applied broadly**: add dashboard URL annotations across multiple alert files (alertmanager, compactor, blocks, ingest-storage, and the main alerts file)

The dashboard URL should include `var-<label>={{ $labels.<label> }}` query parameters for each label in `cluster_labels`, so the dashboard opens pre-filtered to the alert's context.

## Files to Look At

- `operations/mimir-mixin/alerts/alerts-utils.libsonnet` — shared alert utility functions; add the new dashboard URL helpers here
- `operations/mimir-mixin/config.libsonnet` — mixin configuration; add the new URL prefix config field here
- `operations/mimir-mixin/alerts/alerts.libsonnet` — main alerts; apply dashboard URLs to relevant alerts
- `operations/mimir-mixin/alerts/alertmanager.libsonnet` — alertmanager alerts
- `operations/mimir-mixin/alerts/compactor.libsonnet` — compactor alerts
- `operations/mimir-mixin/alerts/blocks.libsonnet` — blocks/store-gateway alerts
- `operations/mimir-mixin/alerts/ingest-storage.libsonnet` — ingest storage alerts

After implementing the code changes, update the project's contributing documentation (`docs/internal/contributing/README.md`) to include formatting guidance for jsonnet/libsonnet files. Contributors working on the mixin should know which `make` targets to run for formatting and building compiled output.
