## Bug

The `.cursor/` version of `skills/adding-a-command/SKILL.md` instructs agents to use **default exports** for command files, which contradicts the pattern used everywhere in `src/commands/` and documented correctly in the `.claude/` and `.agents/` versions:

```typescript
// What .cursor/ skill says (wrong):
export default async (options: any, ctx: CLIContext) => { ... }
import addCommand from './commands/mycommand.js'

// What src/cli.ts expects (correct):
export async function myCommand(options?: OptionType) { ... }
import { myCommand } from './commands/mycommand.js'
```

Every command in the actual codebase (`initCommand`, `statusCommand`, `scoreCommand`, `refreshCommand`, etc.) uses named exports. The `src/cli.ts` imports them all by name.

## Why it matters

An agent following the `.cursor/` skill will generate a default-export command file. When `src/cli.ts` tries to use a named import (`import { myCommand } from ...`), Node.js will throw:

```
SyntaxError: The requested module does not provide an export named 'myCommand'
```

The command will never register or run.
