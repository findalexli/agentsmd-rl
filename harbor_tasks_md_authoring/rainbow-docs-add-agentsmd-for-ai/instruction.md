## What to add / change

AI coding agents work better when they have upfront context about the project's architecture and conventions. Right now each session starts cold and has to rediscover things like which cmds to run for type checking (which my Claude often does wrong 😠), our custom store creators, the domain architecture migration, and the barrel export restriction.

This change adds an `AGENTS.md` at the repo root with the essential context: verification commands, state management system (the three store creators, legacy React Query and Redux), source layout with the migration direction, and key code conventions. A `.claude/CLAUDE.md` file references it so Claude Code also picks it up automatically (shame for not supporting `AGENTS.md` Claude!).

The file content was generated by Claude `/init` and iterated several times with heavy refinements and manual edits to make sure it ONLY includes useful things. The file is intentionally lean following the principle that always-loaded agent context should only cover things that apply to most sessions. Patterns for which this does not apply (worklets, custom navigators, E2E setup, one time project setup instructions etc) are left for agents to discover from the code.
